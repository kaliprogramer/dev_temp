______________________________________________________________
| 🚀 DRF VIEWS (HOW TO WRITE + WHEN + WHY)                 |
|______________________________________________________________|

| 🧠 1. APIView (FULL CONTROL MODE 🔥)                    |
|______________________________________________________________|
|
| 📌 CODE (views.py)
|
| => from rest_framework.views import APIView
| => from rest_framework.response import Response
|
| => class ProductAPI(APIView):
|      def get(self, request):
|          return Response({"msg": "GET request"})
|
|      def post(self, request):
|          return Response({"msg": "POST request"})
|
|------------------------------------------------------------|
|
| 📍 WHEN TO USE:
| 👉 complex logic (login, payments, custom flow)
|
| 💡 WHY:
| 👉 you control EVERYTHING manually
|
|------------------------------------------------------------|

| ⚡ 2. GENERIC VIEWS (FAST + CLEAN 🔥)                    |
|______________________________________________________________|
|
| 📌 CODE
|
| => from rest_framework.generics import ListAPIView
|
| => class ProductListView(ListAPIView):
|      queryset = Product.objects.all()
|      serializer_class = ProductSerializer
|
|------------------------------------------------------------|
|
| 📍 WHEN TO USE:
| 👉 simple GET list or CRUD pages
|
| 💡 WHY:
| 👉 no need to write get() manually
|
|------------------------------------------------------------|

| 🧩 3. MIXINS + GENERICAPI (BUILD YOUR OWN STYLE ⚙️)     |
|______________________________________________________________|
|
| 📌 CODE
|
| => from rest_framework.generics import GenericAPIView
| => from rest_framework.mixins import ListModelMixin
|
| => class ProductView(ListModelMixin, GenericAPIView):
|      queryset = Product.objects.all()
|      serializer_class = ProductSerializer
|
|      def get(self, request):
|          return self.list(request)
|
|------------------------------------------------------------|
|
| 📍 WHEN TO USE:
| 👉 when default generic views are not enough
|
| 💡 WHY:
| 👉 mix and control behavior manually
|
|------------------------------------------------------------|

| 🚀 4. MODELVIEWSET (PRODUCTION KING 👑)                 |
|______________________________________________________________|
|
| 📌 CODE
|
| => from rest_framework.viewsets import ModelViewSet
|
| => class ProductViewSet(ModelViewSet):
|      queryset = Product.objects.all()
|      serializer_class = ProductSerializer
|
|------------------------------------------------------------|
|
| 📍 WHEN TO USE:
| 👉 full CRUD APIs (real apps, startups, production)
|
| 💡 WHY:
| 👉 gives ALL CRUD automatically
|
|------------------------------------------------------------|

| 🌐 5. URL ROUTER (FOR VIEWSETS ONLY)                   |
|______________________________________________________________|
|
| => from rest_framework.routers import DefaultRouter
|
| => router = DefaultRouter()
| => router.register(r'products', ProductViewSet)
|
| => urlpatterns = router.urls
|
|------------------------------------------------------------|

| ⚖️ QUICK DECISION GUIDE                                |
|______________________________________________________________|
|
| 👉 APIView        → complex logic
| 👉 GenericAPIView  → partial control
| 👉 Generic Views   → fast CRUD pages
| 👉 ModelViewSet    → full production API
|
|------------------------------------------------------------|

| 💡 SIMPLE REAL-WORLD EXAMPLE                           |
|______________________________________________________________|
|
| 👉 eCommerce app:
|
| - Product API → ModelViewSet
| - Payment API → APIView
| - Admin lists → Generic Views
|
|______________________________________________________________|
